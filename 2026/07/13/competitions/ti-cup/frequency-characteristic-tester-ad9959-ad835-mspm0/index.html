2013 E 题简易频率特性测试仪：软硬件总体设计

本文根据最新工程 README 重新整理。当前方案不再只是一份器件选型说明，而是以学长提供的 FrequencyResponseTester_v3 参考工程为软件基础，合并题目分析、硬件设计和后续联调计划，作为 2013 E 题“简易频率特性测试仪”的统一开发说明。

文章中特别区分三类状态：已实现表示当前参考代码已经存在并完成 MSPM0G3507 编译链接；已有框架表示接口或数据结构已经有了，但算法和联调还没闭合；后续设计表示完整仪器目标，当前源码尚未实现。

一、方案目标

题目要求仪器在约 1 MHz 到 40 MHz 范围内产生扫频信号，并测量被测网络的幅频和相频特性。当前软件默认参数与题目基本一致：

连续扫频时间 2 s
单频输出幅度设置 1.0 Vpp
标记频率 20 MHz
测量目标 增益、相位、中心频率、-3 dB 带宽

二、总体架构

整机按功能可以拆为五块：

  1. 数字控制板：番茄派 MSPM0G3507，负责状态机、DDS 控制、ADC 采集、计算和显示。
  2. DDS 信号源板：AD9959、25 MHz 参考时钟、PLL、DAC 输出和控制接口。
  3. 射频输出链路：重构低通、OPA695 宽带放大、50 Ω源端匹配和 SMA 接口。
  4. 正交解调板：50 Ω输入端接、两片 AD835、I/Q 本振输入、基带低通与电平转换。
  5. 人机交互部分：串口屏、五向按键、旋转编码器和 RGB 状态灯。

四、AD9959 与射频输出

当前 AD9959 驱动按以下时钟关系配置：

500 MHz 系统时钟下，32 位频率控制字理论分辨率约 0.116 Hz；14 位相位字理论分辨率约 0.022°，都远小于题目要求。

控制接口包括：

  • SCLK：串行时钟；
  • SDIO0：串行数据；
  • CS：片选；
  • IO_UPDATE：让多个通道的新参数同时生效；
  • RESET：硬件复位。

IO_UPDATE 是保证正交关系的关键。正确流程是先写完各通道的频率、幅度和相位寄存器，再统一触发更新，而不是逐通道写完就立即生效。

AD9959 的 DAC 输出不能直接视为理想正弦源。输出端会有采样镜像、时钟泄漏以及随频率变化的幅度包络，因此射频通道需要重构低通。CH0 激励链路建议为：

输出幅度必须在外接 50 Ω负载时测量，不能只看空载电压。源端 50 Ω与负载 50 Ω会形成分压，这一点直接影响题目中以 u_s/2 为参考的增益定义。

五、DUT 接口与 AD835 解调

射频接口建议使用 SMA 和短同轴线，不用杜邦线传输 40 MHz 信号。DUT 输出进入仪器后，只放置一个精密 49.9 Ω端接：

这里有一个非常重要的避坑：不能给两片 AD835 各放一个 50 Ω端接。两个 50 Ω并联会变成 25 Ω，直接改变被测网络的有载增益和 Q 值。

双 AD835 的连接思路是：

  • 两片 AD835 的 X 输入都接 DUT 输出；
  • I 路 Y 输入接 0°本振；
  • Q 路 Y 输入接 ±90°本振；
  • W 输出送入基带低通；
  • Z 输入可接模拟地，也可以在后级完成偏置。

AD835 建议使用低噪声 ±5 V 模拟电源。两片器件应尽量镜像布局，X、Y、W 三类走线保持同环境，电源去耦至少在每片附近放置 100 nF + 4.7 µF。

六、基带低通、偏置和 ADC

AD835 输出包含期望的低频分量和约 2f 的高频乘积项，因此每一路都需要低通滤波：

低通截止频率不应该照搬很低的 1 Hz。1 Hz 虽然看起来“更干净”，但建立时间会让 391 点扫频远远超过 30 s。当前建议初版使用约 100 Hz：

R=15.8 kΩ,C=100 nF fc≈12πRC≈100.7 Hz

时间常数约 1.58 ms，稳定到 0.1% 约需 11 ms。最低输入 1 MHz 时，乘积高频项是 2 MHz，100 Hz 一阶低通已经能提供足够衰减。

MSPM0 ADC 只能接收非负电压，所以 I/Q 基带需要平移到 ADC 量程中部。硬件设计可用 OPA2320 做 4 倍增益和 1.25 V 平移：

VADC≈1.25 V+4.02VBB

ADC 参考采用 MSPM0G3507 内部 2.5 V VREF。外部接口板应在 PA23 / VREF+ 到 GND 放 1.0 µF 电容，并用 2.5 V 参考经 20 kΩ / 20 kΩ 分压和 OPA2320 缓冲产生 1.25 V 中点。

九、频率特性测试状态机

当前参考工程包含四种主模式：

  1. FRT_MODE_CALIBRATION：显示校准页面并管理等待、进行中、完成状态；当前校准算法仍是 TODO。
  2. FRT_MODE_SINGLE_TONE：约每 200 ms 更新一次 DDS，采集一次 I/Q 并计算单频增益和相位。
  3. FRT_MODE_SWEEP：按起点、终点、步进和总时间连续更新 DDS，用于输出扫频和浏览界面。
  4. FRT_MODE_SWEEP_MEASURE：逐点更新 DDS、采样并写入曲线数组，完成后返回扫频页面。

FRT_Param_t 保存运行参数，包括单频频率和幅度、扫频起点终点步进、扫频时间、基带偏置 LPFOffset、逐频点 gainOffset[] 和 phaseOffset[]。

FRT_Data_t 保存测量结果，包括单频增益和相位、扫频曲线数组、当前标记点数据、中心频率 centerF 和带宽 bandwidth。目前中心频率和带宽字段已经存在，但自动提取算法还没有完成。

当前扫频数据缓冲区长度为 401，默认 1 MHz 到 40 MHz、100 kHz 步进实际使用 391 个频点，保留了少量余量。

十一、ADC 与 DMA 后续方向

当前 ADC 工作流程是：

当前代码是两路分别启动、每个频点各读取一次，不是严格同步采样，也没有 DMA 和多次平均。早期联调可以先这样跑通链路；要稳定达到幅相精度，后续应升级为：

  1. 同一个定时器事件触发 ADC0、ADC1；
  2. DMA 采集 64 到 256 组 I/Q；
  3. 丢弃换频后的过渡样本；
  4. 对剩余数据做均值、异常值检查和超时处理；
  5. 幅值过小时冻结相位，避免低信噪比点乱跳。

十二、校准体系

校准是达到幅度和相位指标的核心，不能只依赖器件理论参数。

当前数据结构已经为每个频点预留：

最小可用方案：

  1. 关闭 DDS 或断开射频，测量 I/Q 静态偏置，更新 LPFOffset 或分别保存 I/Q 偏置。
  2. 使用 50 Ω直通连接，逐频点测量增益和相位。
  3. 把理想直通值与实测值之差写入 gainOffset[]、phaseOffset[]。
  4. DUT 测量时按频点查表修正。

更稳健的方案是保存直通复数：

如果还要修正 I/Q 正交角误差，可以进一步保存每个频点的 2×2 矩阵：

[Imeas−bIQmeas−bQ]=[m11m12m21m22][Re(H)Im(H)]

运行时使用矩阵逆变换恢复真实复数响应。这个方案精度更高，但需要新增校准数据结构、Flash 保存和矩阵运算，属于后续设计。

逐频点校准非常必要。40 MHz 时，1 ns 路径延时就对应约 14.4°相移。如果只在一个频点校准，其他频点仍会因为线缆、滤波器和 PCB 路径延时产生明显相位误差。

十三、RLC 被测网络

题目典型网络中心频率约为 20 MHz，有载 Q 约为 4，则理论带宽：

若采用 C = 18 pF，理论电感约为：

L=1(2πf0)2C≈3.52 μH

实际选型不能只看标称值。20 MHz 下，电感自谐振频率、实际 Q 值、等效串联电阻、焊盘寄生和电容 ESR 都会影响结果。电容优先使用 C0G/NP0。

软件从扫频数组提取参数的建议流程：

  1. 查找最大增益点，得到中心频率 f0；
  2. 计算峰值以下 3 dB 的目标电平；
  3. 从峰值向左右搜索两个交点，并线性插值得到 fL、fH；
  4. 计算 BW = fH - fL；
  5. 计算 Q = f0 / BW。

当前 FRT_Data_t 已有 centerF 和 bandwidth 字段，但自动提取算法还没有在参考代码中完成。

十六、联调顺序

建议坚持分模块闭环，避免一次性连接整机后难以定位问题。

阶段 A：数字与 DDS

  1. 验证 MSPM0 80 MHz 运行和 1 ms SysTick；
  2. 示波器检查 SCLK、SDIO0、CS、RESET、IO_UPDATE；
  3. 验证 AD9959 在 1 MHz、20 MHz、40 MHz 输出正确；
  4. 验证两通道同频和 90°相位差；
  5. 验证 100 kHz 步进和 2 s 连续扫频。

阶段 B：单路 AD835

  1. 只安装 I 路乘法器和低通；
  2. 输入同频同相信号，确认得到稳定直流；
  3. 改变相位为 0°、90°、180°，验证余弦关系；
  4. 测量无射频时的输出偏置和温漂。

阶段 C：双路 I/Q

  1. 增加 Q 路并检查两路布局、增益和偏置一致性；
  2. 验证 atan2(Q,I) 能正确覆盖四象限；
  3. 对已知相移和衰减器做点频误差统计；
  4. 再决定是否加入本振缓冲或可切换基带增益。

阶段 D：采集与校准

  1. 从单次 ADC 读取升级为定时器同步触发；
  2. 加入 DMA、多次平均和过渡样本丢弃；
  3. 完成零偏、直通和逐频点校准；
  4. 测试更换频点后的最佳等待时间。

阶段 E：扫频与显示

  1. 完成 391 点测量和曲线显示；
  2. 完成标记点、峰值、中心频率和 -3 dB 带宽；
  3. 与示波器、信号源或网络分析仪逐点对比；
  4. 最后处理温漂、重复装配和长时间运行问题。

十七、风险与避坑

HMI 阻塞发送 扫频节拍被拖慢 扫频时降低刷新率或改为缓冲发送

最重要的工程避坑可以压缩成六句话：

  1. MSPM0 ADC 不能直接测 40 MHz 射频，只采 AD835 解调后的低速基带。
  2. 不能使用两个 50 Ω并联端接同一个 DUT 输出。
  3. 不能只校准单一频点。
  4. 不能只用一次 ADC 采样作为最终测量值。
  5. 不能让串口屏每个频点大量刷新。
  6. 不能在尚未验证单路 AD835 前直接制作全集成板。

十九、小结

这一版方案的重点不是“又换了一个器件组合”，而是把目标、代码现状和后续硬件设计放到同一张图里：

  • AD9959 提供相干 DDS 激励和 I/Q 本振；
  • OPA695 负责射频激励链路的宽带放大和隔离；
  • AD835 负责把 40 MHz 以内射频响应搬到低速 I/Q 基带；
  • MSPM0G3507 负责 DDS 控制、ADC 采集、状态机、校准计算和 HMI；
  • 当前参考工程已经具备 AD9959、ADC、UI、按键、编码器和 HMI 框架；
  • 真正还要攻下的是双 AD835 模拟前端、同步 DMA 平均、逐频点校准、中心频率和 Q 值提取。

下一步最务实的路线是：先把 DDS、单路 AD835、双路 I/Q 和 ADC 采集分阶段跑通，再做 391 点完整测量。只要每一段都能单独闭环，最后整机就不会变成一团难以定位的问题。
