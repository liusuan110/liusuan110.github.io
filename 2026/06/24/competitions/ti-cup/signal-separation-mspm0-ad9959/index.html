用「再生法」做电赛信号分离：MSPM0 + AD9959 的软硬件架构

这是 2023 年电赛 H 题「信号分离装置」的一次完整工程实现记录。输入是一路混在一起的信号 C=A+B ，装置要把 A 、 B 各自的频率、幅度、波形（正弦 / 三角）认出来、重建出来，还要测它们的相位差。

整套方案的骨架是 再生法 + 闭环锁相：不去硬算"减法"，而是用 DDS 造出副本，再用闭环把副本锁到原信号上。本文从硬件讲到软件，重点是两条锁相路线（数字 PLL / 模拟鉴相 + PID）和让算法能在电脑上离板验证的工程架构。

平台：TI MSPM0G3507（番茄派）+ AD9959 四通道 DDS + 淘晶驰串口屏。

一、题目与总思路：什么是"再生法"

信号分离最直觉的想法是"做减法"——但你手里只有混合信号 C ，没有任何一路的纯净参考，硬减无从下手。

换个思路：与其把两路减开，不如照着它们各画一张一模一样的副本。

  1. 先分析 C ，量出 A 、 B 各自的频率、幅度、波形；
  2. 用 DDS 按这些参数再生出副本 A′ 、 B′ ；
  3. 用一个闭环不停校准副本，让它和原信号分毫不差。

一旦副本画得和原件一模一样，你自然就把每一路单独"分离"出来了——这就是再生法。它把"信号分离"这个难题，转化成了"参数测量 + 闭环跟踪"两个可解的子问题。

至于"为什么副本需要不停校准"，看完硬件就懂了。

二、硬件架构

2.1 三个角色

  • 大脑：MSPM0G3507——Cortex-M0+ @80 MHz，双 12 位 ADC，带硬件数学加速器 MATHACL（M0+ 没有 FPU，定点三角运算靠它撑），128 KB Flash / 32 KB SRAM。
  • 信号发生器：AD9959——四通道 DDS，系统时钟 500 MHz。给它一个频率字 FTW，它就吐出对应频率的正弦。副本 A′ 、 B′ 就是它生成的，SPI 控制。 FTW=round(f×232fsys),fsys=500 MHz
  • 人机界面：淘晶驰串口屏——UART 收 ASCII 指令显示结果。

采集分工：ADC0 收混合信号 C ，ADC1 收反馈。

2.2 关键难点：三个时钟域 → 必须闭环

这是整个架构的"病根"，也是它比一般方案硬核的地方：

信号源、MSPM0、AD9959 各有各的晶振，三个时钟域互相独立。

哪怕 FTW 算得再准，副本 A′ 的频率和真 A 也会差一丢丢（亚 Hz 级），再加上晶振 ppm 级偏差， A′ 的相位会相对真 A 缓慢漂移，越积越大。开环（只灌一次 FTW）绝对锁不住。

所以必须有闭环实时测、实时修——这就是后面要花大篇幅讲的锁相环。

对照：很多 STM32 参考方案把 DAC 和 ADC 放在同一个时钟域，不存在漂移，就不需要这套闭环。三时钟域是本方案复杂度的根源，也是它的技术含量所在。

2.3 双 ADC 定时器同步

发挥部分要测相位差，这就要求两路 ADC 同时刻采样。如果用"软件先启动 ADC0、再启动 ADC1"的方式，两者差几条指令（约百纳秒），会给相位差引入固定偏差。

解决办法：让 ADC0 / ADC1 订阅同一个定时器事件，由硬件同时触发——

两路采样落在同一时钟沿，相位差测量就干净了。

三、信号处理链：数据的一趟旅程

几个工程细节：

  • 加汉宁窗：信号周期与采样窗口对不齐时，频谱会泄漏，一根峰糊到相邻 bin。先加窗再 FFT，把泄漏压下去，离 bin 的幅度/谐波才量得准。
  • q15 定点 FFT：M0+ 没有 FPU，用 CMSIS-DSP 的 arm_cfft_q15（1024 点）。为了不让小谐波被量化噪声淹没，FFT 前先对数据块归一化吃满 q15 量程。
  • 相位提取：在已知 bin 上对复数取 atan2(im,re) （用 MATHACL 加速），供锁相环使用。

四、怎么从频谱里判正弦还是三角

波形的"指纹"不在基频，在谐波：

  • 纯正弦：能量只在基频 f0 一根线，没有谐波。
  • 三角波：基频 + 奇次谐波（ 3f0,5f0,7f0… ），幅度按 1/n2 衰减。

各次谐波相对基频的幅度：

谐波 正弦 三角 (1/n2) 方波 (1/n)
3 次 ≈0 11.1 % 33 %
5 次 ≈0 4.0 % 20 %
7 次 ≈0 2.0 % 14 %

判别配方：找到基频 bin k0 ，量 3k0 、 5k0 处的幅度，算比值 mag[3k0]/mag[k0] ——接近 0 是正弦，接近 0.11 是三角，取个中间阈值一刀切，再用 5 次谐波交叉验证。

一个阴险的坑：谐波重叠

如果 B 的基频正好落在 A 的三次谐波上（ fB=3fA ），那么 3k0 这根峰就是 B 的基频 + A 的三次谐波叠在一起：

  • 量 B 的幅度时，要把 A 的三次谐波贡献（若 A 是三角，约 mag[k0]/9 ）减掉；
  • 判 B 的波形时， B 的三次谐波在 9k0 ，又撞上 A 的九次谐波（约 mag[k0]/81 ），也得减掉。

不做这层"去污染"， B 就会被误判。这是信号分离里最容易翻车、也最值得细抠的地方。

五、锁相：两条路线，一个开关切换

锁相是把副本"钉死"在原信号上的核心。本方案做了两条独立实现，用一个编译开关二选一，便于上板 A/B 对比：

5.1 路线 A：II 型数字 PLL

思路：ADC1 采反馈 → FFT → atan2 提相位 → 数字环路滤波器 → 调 FTW。

几个定点技巧：

  • ① 相位用 int32， 232=360° ，相减自然回绕：比如目标 350°、反馈 10°，朴素相减得 +340°（绕远路），但 int32 补码自动回绕成 −20°（最短路径）——硬件免费帮你处理了过 360° 回绕，不用写 if (>180) -=360。
  • ② 积分器用 int64 全分辨率，先累加后移位：如果每拍"先右移再累加"，比 227 小的误差移完就成 0，永远累不进去（死区）。用 int64 全分辨率累加、读出时才移位，哪怕误差为 1 也在慢慢攒，最终翻过门限——这才能做到零稳态误差。
  • 为什么是 II 型：DDS 的 NCO 本身是一个积分器（频率积分成相位），环路滤波器再加一个积分项 → 两个积分器 = II 型。只有 II 型能在恒定频差下把稳态相位误差压到零——而三时钟域之间恰恰是恒定频差。这不是炫技，是被需求逼出来的（否则相位差就测不准）。

环路增益按 5 ms 控制周期整定（离板仿真验证过：周期拖到 50 ms 会发散），所以前面才要用定时器把节奏卡死。

5.2 路线 B：模拟鉴相 + PID

把鉴相从软件搬到模拟硬件：

外部模拟乘法器把 原信号 × AD9959 正交本振 相乘，低通后得到一个直流电压。两个同频正弦相乘：

sin⁡(ωt+φs)sin⁡(ωt+φlo)=12[cos⁡(φs−φlo)−cos⁡(2ωt+…)]

滤掉 2 倍频，直流项 ∝cos⁡(Δφ) ，在两者相差 90°（正交）时为零。

于是：ADC1 读这个电压当误差，目标设 0（正交锁定），float PID 微调 DDS 频率把相位拉到正交。锁定点选 90° 是因为乘法器鉴相器在那里最线性，误差信号最干净。

巧妙的地方是 AD9959 四个通道全用上：CH0/CH1 输出 A′ / B′ ，CH2/CH3 当正交本振喂给外部乘法器做鉴相（相位字 4096=90° ，发挥②的相位差就靠给本振叠一个已知偏移读出来），四通道用一次 IOUpdate 原子更新。

PID 用 float 而不是定点：M0+ 虽无 FPU，但 PID 每控制周期才算两路、不逐采样，软件浮点开销可接受，换来代码清爽。增益还做了按频率调度——基准增益在 50 kHz 处标定，实际增益 =基准×(f/50kHz) ，让全频段环路动态一致。

5.3 两条路线对比

路线 A：数字 PLL（默认） 路线 B：模拟鉴相 + PID
鉴相方式 ADC1 块采 → FFT → atan2 外部乘法器（信号 × 正交本振）→ 直流电压
反馈量 数字相位（Q31） 鉴相直流电压（ADC 读）
控制器 II 型数字 PLL（定点 / 移位增益） float PID（增益按频率调度）
锁定点 相位误差 → 0 正交（90°），鉴相电压 → 偏置
AD9959 CH0/1 输出 CH0/1 输出 + CH2/3 正交本振
复杂度落点 全在固件 挪到模拟硬件，固件更简单
额外硬件 无 需模拟乘法器 + 低通
RAM 占用 较高（需反馈采集缓冲） 省约 2 KB（只读两个直流电压）

两条路并存的好处是对冲：上板时都烧进去，哪条锁得稳留哪条；数字路无需额外模拟器件、已离板验证，作默认；模拟路把鉴相甩给硬件、固件更轻，作实验分支。

六、软件工程：core / glue 分离 + 离板测试

这套代码改起来不慌，靠的是一条贯穿始终的原则：把"聪明的算法"和"碰硬件的代码"切开。

每个模块都拆成两个文件：

  • 核（*_core.c，纯 C）：所有算法。只 #include <stdint.h>，不碰寄存器、不调厂商库、不用 printf/浮点格式化、无硬件副作用。
  • 胶水（无 _core）：把核接到真实硬件——ADC+DMA、写 AD9959 寄存器、发 UART。要薄。

以 PLL 为例，整个锁相的"脑子"在 pll_core.c，而它根本不知道 AD9959 的存在；胶水层只做"翻译 + 调核 + 落地"：

因为核是纯 C，电脑能直接编译运行它。 这就是 host_test/——在 PC 上用 gcc 编译、跑 assert，不需要板子、不需要 MCU 的 SDK：

测试里可以凭空造出"三时钟域"：让仿真的源相位和副本相位以略不同的频率推进，看 PLL 多少拍把误差收敛到零；扫增益、灌不同频差、跑多种场景——全在电脑上秒级反馈。"周期 5 ms 稳、50 ms 发散"这个结论，就是这么发现的，没烧一次板子。

同一份核源码被两个编译器各编一次：tiarmclang 进固件，gcc 进测试。这套拆法买来的是：算法 bug 在上电前就抓出来、改完可回归、换平台只动薄胶水层。最妙的是 host_test 本身就是"纯度看门狗"——谁手滑让核依赖了硬件头文件，gcc 那条路立刻编不过。

分析、锁相、PID、遥测、串口屏五个模块都严格守这套三件套（核 + 胶水 + 测试）。

七、上板：可观测性与人机界面

算法对不对在电脑上能验，但"板子上到底发生了什么"得有眼睛：

  • 遥测：每个控制周期把内部状态（各路 ADC 原始 min/mean/max、测得频率、相位、PLL 误差、是否锁定、当前阶段）用 UART 吐出来。M0+ 没 FPU，所以自己写整数→ASCII、Q31 角→十分之一度，不碰 printf/%f，省 Flash 又避免浮点格式化。上板调试时这就是听诊器。
  • 分阶段 bring-up：上电不是一把全开，而是分 4 个阶段逐步点亮（采集 → 分析 → 再生 → 闭环），每步用遥测确认无误再进下一步。
  • 淘晶驰串口屏：显示两路的频率、幅度、波形、相位差、锁定状态。指令构建做了增量刷新——只发和上一帧不同的字段，锁定稳定后几乎不发，把串口带宽占用压到最低。

八、小结

一句话画像：

硬件：双 ADC 同步采集 → MCU 分析 → DDS 再生 → 闭环锁定 的再生法装置，核心难点是三时钟域漂移逼出来的锁相环。 软件：纯 C 算法核 + 薄硬件胶水 + 离板测试 的可验证三明治，锁相提供数字 PLL / 模拟鉴相+PID 两条可切换路线。 角色：MSPM0 是大脑，AD9959 是嘴，串口屏是脸，遥测是听诊器。

目前算法层（FFT 分析、波形判别、谐波去污、数字 PLL、PID 数学）都已离板验证；定时器同步、模拟鉴相器与电压标定、串口屏这些硬件相关项要上板才能最终确认。把"上电前能锁死的正确性"尽量在电脑上锁死，剩下的交给板子——这是整套架构想达到的状态。
