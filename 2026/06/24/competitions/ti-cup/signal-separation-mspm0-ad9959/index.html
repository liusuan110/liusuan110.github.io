用「再生法」做电赛信号分离：MSPM0 + AD9959 的软硬件架构

这是 2023 年电赛 H 题「信号分离装置」的一次完整工程实现记录。输入是一路混在一起的信号 C=A+B ，装置要把 A 、 B 各自的频率、幅度、波形（正弦 / 三角）认出来、重建出来，还要测它们的相位差。

整套方案的骨架是 再生法 + 闭环锁相：不去硬算"减法"，而是用 DDS 造出副本，再用闭环把副本锁到原信号上。锁相主线是「模拟鉴相 + PID」（把鉴相交给一颗模拟乘法器），全数字 PLL 作替代——本文重点讲透主线，连模拟前端怎么搭都给了硬件同学。

平台：TI MSPM0G3507（番茄派）+ AD9959 四通道 DDS + 淘晶驰串口屏。

一、题目与总思路：什么是"再生法"

信号分离最直觉的想法是"做减法"——但你手里只有混合信号 C ，没有任何一路的纯净参考，硬减无从下手。

换个思路：与其把两路减开，不如照着它们各画一张一模一样的副本。

  1. 先分析 C ，量出 A 、 B 各自的频率、幅度、波形；
  2. 用 DDS 按这些参数再生出副本 A′ 、 B′ ；
  3. 用一个闭环不停校准副本，让它和原信号分毫不差。

一旦副本画得和原件一模一样，你自然就把每一路单独"分离"出来了——这就是再生法。它的精髓是把"无中生有地拆信号"这个难题，换成两个有现成工具的子问题：

  • 测量：从 C 的频谱里读出每一路的频率 / 幅度 / 波形——FFT 能干（第三、四节）；
  • 跟踪：让 DDS 副本始终贴住原信号——闭环锁相能干（第五节）。

一条贯穿全文的思路线。 这道题真正的难点不在"测量"，而在"跟踪"。下面三句话像多米诺骨牌，把后面每一个设计都逼了出来：

  1. 副本一定会漂。 信号源、MCU、DDS 三个晶振互相独立（第二节），副本 A′ 和真 A 的频率永远差一丢丢、相位越积越偏。→ 所以必须闭环，开环灌一次 FTW 锁不住。
  2. 闭环的命门，是每个控制周期都能拿到"副本与原信号的相位误差"。 拿到误差，剩下的就是个标准控制问题（PI / PID）。→ 于是全文的核心问题收敛成一句话：这个相位误差，到底怎么测？
  3. 这个误差怎么测，决定了整套方案的形态。 我们的主线是模拟鉴相——一颗乘法器把原信号和正交本振相乘，直接输出一个正比于相差的电压（第五节）；全数字（对反馈做 FFT 提相位）作替代。

记住这条线（漂 → 必须闭环 → 闭环靠拿到相位误差 → 误差有两种拿法），后面每一节都是在回答它的某一环。

二、硬件架构

2.1 三个角色

  • 大脑：MSPM0G3507——Cortex-M0+ @80 MHz，双 12 位 ADC，带硬件数学加速器 MATHACL（M0+ 没有 FPU，定点三角运算靠它撑），128 KB Flash / 32 KB SRAM。
  • 信号发生器：AD9959——四通道 DDS，系统时钟 500 MHz。给它一个频率字 FTW，它就吐出对应频率的正弦。副本 A′ 、 B′ 就是它生成的，SPI 控制。 FTW=round(f×232fsys),fsys=500 MHz
  • 人机界面：淘晶驰串口屏——UART 收 ASCII 指令显示结果。

采集分工：ADC0 收混合信号 C ；主线（模拟鉴相）下 ADC1 读两路鉴相电压，数字替代时改收 A′+B′ 反馈。

2.2 关键难点：三个时钟域 → 必须闭环

这是整个架构的"病根"，也是它比一般方案硬核的地方：

信号源、MSPM0、AD9959 各有各的晶振，三个时钟域互相独立。

哪怕 FTW 算得再准，副本 A′ 的频率和真 A 也会差一丢丢（亚 Hz 级），再加上晶振 ppm 级偏差， A′ 的相位会相对真 A 缓慢漂移，越积越大。开环（只灌一次 FTW）绝对锁不住。

所以必须有闭环实时测、实时修——这就是后面要花大篇幅讲的锁相环。

对照：很多 STM32 参考方案把 DAC 和 ADC 放在同一个时钟域，不存在漂移，就不需要这套闭环。三时钟域是本方案复杂度的根源，也是它的技术含量所在。

2.3 双 ADC 定时器同步

说明：下面这条"双 ADC 同步"主要服务数字替代路线（块采 A′+B′ 反馈）；主线模拟鉴相靠正交本振的相位偏移测相位差（见 5.2），ADC1 只单次读电压。而 ADC0 的定时器触发（锁 fs ）两条路都用。

发挥部分要测相位差，这就要求两路 ADC 同时刻采样。如果用"软件先启动 ADC0、再启动 ADC1"的方式，两者差几条指令（约百纳秒），会给相位差引入固定偏差。

解决办法：让 ADC0 / ADC1 订阅同一个定时器事件，由硬件同时触发——

两路采样落在同一时钟沿，相位差测量就干净了。

三、信号处理链：数据的一趟旅程

几个工程细节：

  • 加汉宁窗：信号周期与采样窗口对不齐时，频谱会泄漏，一根峰糊到相邻 bin。先加窗再 FFT，把泄漏压下去，离 bin 的幅度/谐波才量得准。
  • q15 定点 FFT：M0+ 没有 FPU，用 CMSIS-DSP 的 arm_cfft_q15（1024 点）。为了不让小谐波被量化噪声淹没，FFT 前先对数据块归一化吃满 q15 量程。
  • 相位提取：在已知 bin 上对复数取 atan2(im,re) （用 MATHACL 加速），供锁相环使用。

四、怎么从频谱里判正弦还是三角

波形的"指纹"不在基频，在谐波：

  • 纯正弦：能量只在基频 f0 一根线，没有谐波。
  • 三角波：基频 + 奇次谐波（ 3f0,5f0,7f0… ），幅度按 1/n2 衰减。

各次谐波相对基频的幅度：

谐波 正弦 三角 (1/n2) 方波 (1/n)
3 次 ≈0 11.1 % 33 %
5 次 ≈0 4.0 % 20 %
7 次 ≈0 2.0 % 14 %

判别配方：找到基频 bin k0 ，量 3k0 、 5k0 处的幅度，算比值 mag[3k0]/mag[k0] ——接近 0 是正弦，接近 0.11 是三角，取个中间阈值一刀切，再用 5 次谐波交叉验证。

一个阴险的坑：谐波重叠

如果 B 的基频正好落在 A 的三次谐波上（ fB=3fA ），那么 3k0 这根峰就是 B 的基频 + A 的三次谐波叠在一起：

  • 量 B 的幅度时，要把 A 的三次谐波贡献（若 A 是三角，约 mag[k0]/9 ）减掉；
  • 判 B 的波形时， B 的三次谐波在 9k0 ，又撞上 A 的九次谐波（约 mag[k0]/81 ），也得减掉。

不做这层"去污染"， B 就会被误判。这是信号分离里最容易翻车、也最值得细抠的地方。

五、锁相：主线「模拟鉴相 + PID」

回到第一节那个收敛后的核心问题：每个控制周期，怎么拿到副本与原信号的相位误差？ 锁相环里"环路滤波器 → 调频率"那半截是标准控制理论，真正决定方案长相的，是误差从哪来。

我们做了两条后端，用一个编译开关二选一；主线走「模拟鉴相 + PID」（把鉴相交给一颗模拟乘法器），全数字 PLL 作为可一键切回的替代：

主循环骨架——采集 → 分析 → 建链 → 闭环 → 显示——不变，只有"闭环"那一环按开关整体换掉。下面先把主线讲透（含硬件怎么搭，给负责硬件的同学），再附上替代的数字路线。

5.1 主线原理：乘法器鉴相 + 正交锁定

核心一招：把"鉴相"从软件搬到模拟硬件。 反馈不再需要对一整块反馈信号做 FFT，只要读两个直流电压：

一颗外部模拟乘法器把 原信号 × AD9959 正交本振 相乘，低通后得到一个直流电压。两个同频正弦相乘：

sin⁡(ωt+φs)sin⁡(ωt+φlo)=12[cos⁡(φs−φlo)−cos⁡(2ωt+…)]

滤掉 2 倍频，直流项 ∝cos⁡(Δφ) ，在两者相差 90°（正交）时为零。

于是闭环是：ADC 读这个鉴相电压当误差，目标设"正交"（电压 = 偏置），float PID 微调 DDS 频率，把相位拉到正交锁定。锁定点选 90°，是因为乘法器鉴相器在那里最线性、误差信号最干净。

增益做了按频率调度——基准增益在 50 kHz 处标定，实际增益 =基准×(f/50kHz) ，让全频段环路动态一致。

一个巧思：起振频率不是瞎猜的——先用 FFT 分析 C 得到每路粗频率（让本振一上来就落在捕获带内），再交给模拟鉴相做细相位锁。FFT 管"找得到"，模拟环管"锁得准"。

5.2 硬件视角：模拟前端怎么搭（重点）

这条路线把复杂度从固件挪到了模拟硬件，所以接线和标定决定成败。给硬件同学的连接图：

AD9959 四个通道这样分：

通道 用途 频率 相位
CH0 / CH1 输出副本 A′ / B′ 跟踪 fA / fB 基准 0
CH2 / CH3 正交本振（喂乘法器鉴相） 同 fA / fB 90°（+ 发挥②偏移）

四通道用一次 IOUpdate 原子生效，频率由 PID 一起微调。一个反直觉但关键的点：把本振锁成与输入正交（90°），同时让输出比本振落后 90°，输出 A′ 就正好和输入 A 同相——这就是"锁本振、出副本"。

接线 / 调理三个要点：

  1. C 要分两路：一路进 ADC0（分析），一路进两个乘法器（鉴相）。别只接一处。
  2. 乘法器输出要 LPF + 抬偏置：番茄派 ADC 是 0–3.3 V 单端，鉴相直流要先低通（去 2 倍频和拍频），再抬到中点 1.65 V（正交零点）才进 ADC1。
  3. 本振电平按乘法器输入范围调（CH2/CH3 幅度在 AD9959 里可设）。

上板要标定的旋钮（全是现场拧的）： 1.65V 鉴相零点、PID 的 Kp/Ki 、增益调度系数、锁定阈值、本振 90° 相位。

两个离板看不见、最容易翻车的点（硬件同学尤其注意）：

  • 环路符号：增频到底把相差拉近还是推远，取决于乘法器输出极性和工作点——符号反了环路会发散而不是锁定。强烈建议留一个"增益符号一键翻转"开关，锁不住先翻它，别急着怀疑别处。
  • LPF 带宽 vs 拍频 fB−fA ：用混合 C 乘本振提某一路相差时，另一路会落在差频 fB−fA 。两路频率靠太近时，差频会漏过低通、污染直流 → 锁不稳。LPF 截止要低到压住最小频率间隔的拍频，但别太低拖慢环路。

5.3 替代路线：II 型数字 PLL（备份）

万一模拟前端一时调不通，可一键切回全数字 PLL（已离板验证）：ADC1 块采反馈 → FFT → atan2 提相位 → 数字环路滤波器 → 调 FTW。

要点：相位用 int32（ 232=360° ，相减自然回绕取最短路径）；积分用 int64 全分辨率（消死区 → 零稳态误差）；II 型（NCO 自带一个积分器 + 环路再加一个）才能在恒定频差下零稳态相差。代价是反馈每拍要多一块采集 + 一次 FFT——这正是主线用模拟鉴相想省掉的那段。

5.4 两条路线对比

主线 路线 B：模拟鉴相 + PID 替代 路线 A：数字 PLL
鉴相方式 外部乘法器（信号 × 正交本振）→ 直流电压 ADC1 块采 → FFT → atan2
反馈量 鉴相直流电压（ADC 读） 数字相位（Q31）
控制器 float PID（增益按频率调度） II 型数字 PLL（定点 / 移位增益）
锁定点 正交（90°），鉴相电压 → 偏置 相位误差 → 0
AD9959 CH0/1 输出 + CH2/3 正交本振 CH0/1 输出
复杂度落点 挪到模拟硬件，固件更轻 全在固件
额外硬件 需模拟乘法器 + 低通 无
每拍负担 只读 2 个直流电压 一块采集 + 一次 FFT
验证状态 PID 数学已离板验证；鉴相物理须上板标定 已离板验证

5.5 为什么主线押模拟，而不是全数字

固件视角看，模拟鉴相把闭环里最重的"第二次采集 + 第二次 FFT"整段拿掉，复杂度挪到一颗便宜的乘法器上——固件更轻、控制周期更宽裕、还省下约 2 KB 反馈缓冲，也有上探更高频率的余地；加上这条思路有学长验证过的实现垫底。代价很明确：正确性的最后一公里在模拟硬件上（乘法器、本振、偏置、锁定阈值都得上板标），所以 5.2 把接线和那两个"翻车点"单列了出来。

也正因为算法早拆成了"纯 C 核 + 薄胶水"（第六节），换锁相后端这种大改，动的几乎全是薄胶水层和 #if 分发，算法核基本没碰——两条路线才能这样自由切换、对冲风险。

六、软件工程：core / glue 分离 + 离板测试

这套代码改起来不慌，靠的是一条贯穿始终的原则：把"聪明的算法"和"碰硬件的代码"切开。

每个模块都拆成两个文件：

  • 核（*_core.c，纯 C）：所有算法。只 #include <stdint.h>，不碰寄存器、不调厂商库、不用 printf/浮点格式化、无硬件副作用。
  • 胶水（无 _core）：把核接到真实硬件——ADC+DMA、写 AD9959 寄存器、发 UART。要薄。

以 PLL 为例，整个锁相的"脑子"在 pll_core.c，而它根本不知道 AD9959 的存在；胶水层只做"翻译 + 调核 + 落地"：

因为核是纯 C，电脑能直接编译运行它。 这就是 host_test/——在 PC 上用 gcc 编译、跑 assert，不需要板子、不需要 MCU 的 SDK：

测试里可以凭空造出"三时钟域"：让仿真的源相位和副本相位以略不同的频率推进，看 PLL 多少拍把误差收敛到零；扫增益、灌不同频差、跑多种场景——全在电脑上秒级反馈。"周期 5 ms 稳、50 ms 发散"这个结论，就是这么发现的，没烧一次板子。

同一份核源码被两个编译器各编一次：tiarmclang 进固件，gcc 进测试。这套拆法买来的是：算法 bug 在上电前就抓出来、改完可回归、换平台只动薄胶水层。最妙的是 host_test 本身就是"纯度看门狗"——谁手滑让核依赖了硬件头文件，gcc 那条路立刻编不过。

分析、锁相、PID、遥测、串口屏五个模块都严格守这套三件套（核 + 胶水 + 测试）。

七、上板：可观测性与人机界面

算法对不对在电脑上能验，但"板子上到底发生了什么"得有眼睛：

  • 遥测：每个控制周期把内部状态（各路 ADC 原始 min/mean/max、测得频率、相位、PLL 误差、是否锁定、当前阶段）用 UART 吐出来。M0+ 没 FPU，所以自己写整数→ASCII、Q31 角→十分之一度，不碰 printf/%f，省 Flash 又避免浮点格式化。上板调试时这就是听诊器。
  • 分阶段 bring-up：上电不是一把全开，而是分 4 个阶段逐步点亮（采集 → 分析 → 再生 → 闭环），每步用遥测确认无误再进下一步。
  • 淘晶驰串口屏：显示两路的频率、幅度、波形、相位差、锁定状态。指令构建做了增量刷新——只发和上一帧不同的字段，锁定稳定后几乎不发，把串口带宽占用压到最低。

八、信号处理知识点（结合本题）

把这道题用到的信号处理"地基"集中梳理一遍——每条都标注本题用在哪，方便对照前文。

8.1 采样定理与频率分辨率

  • 奈奎斯特采样定理：无失真还原信号需 fs≥2fmax ；超过 fs/2 的成分会混叠折回低频，所以采样前应有抗混叠低通。
  • 频率分辨率： N 点 FFT 的 bin 间隔 Δf=fs/N=1/T （ T 为采样总时长）。想分辨更近的两个频率，只能拉长采样时间，与采样率无关。本题 N=1024 。
  • 量化动态范围： b 位 ADC 理论 SNR≈6.02b+1.76 dB ，12 位约 74 dB。
  • 本题用到：决定能测多高频率、频率分得多细；也是后面要做块归一化撑动态范围的根由。

8.2 DFT / FFT 与频谱泄漏

  • DFT： X[k]=∑n=0N−1x[n]e−j2πkn/N ，bin k 对应频率 kfs/N 。实信号频谱共轭对称，只需前 N/2 。
  • 频谱泄漏：非整周期截断等价于乘了矩形窗，频谱被 sinc 卷积 → 能量散到旁瓣，把邻近的峰 / 谐波糊住。
  • 加窗：用汉宁窗压旁瓣（矩形窗旁瓣 −13dB ，汉宁 −31dB ），代价是主瓣变宽——典型的"旁瓣 ↔ 主瓣"折中。
  • 本题用到：FFT 前加汉宁窗，让谐波比、相位读数不被泄漏污染（第三节）。

8.3 栅栏效应与频率插值

  • 栅栏效应：真实频率落在两个 bin 之间时，峰值 bin 的频率有最大 ±Δf/2 误差。
  • 插值估计：用峰值 bin 与左右邻居做抛物线 / 重心 / 比值法插值，把频率精度提到远小于一个 bin（本题约 4.5×）。
  • 本题用到：测频后把结果吸附到题目频率网格（snap-to-grid），配合插值，频率测得准且稳。

8.4 傅里叶级数与波形识别

  • 周期信号 = 基波 + 谐波，各波形的谐波"指纹"不同：
    • 正弦：只有基波；
    • 三角波：奇次谐波，幅度 ∝1/n2 （ 3 次 19 、 5 次 125 ）；
    • 方波：奇次谐波，幅度 ∝1/n （衰减更慢）。
  • 本题用到：第四节靠 3 次谐波占比分开正弦 / 三角；以及 fB=3fA 时的谐波重叠去污。

8.5 相位与相位差测量

  • bin 相位： φ[k]=atan2(ImX[k], ReX[k]) 。
  • 相位差：两路在同一 bin 的相位相减；相位有 2π 周期性，须按最短路径回绕（即第五节的 int32 自然回绕）。
  • 同步采样：两路须同时刻采样，否则采样时刻差 Δt 会引入固定相差 2πfΔt 。
  • 本题用到：发挥②的相位差测量 + 双 ADC 定时器硬同步（第二节）消除 Δt 偏差。

8.6 锁相环与采样控制

  • PLL 三件套：鉴相器（PD）+ 环路滤波器（LF）+ 压控振荡器（VCO，本题即 NCO = DDS 的 FTW）。
  • 环路类型与稳态误差：开环积分器个数 = 型号。NCO 自带一个积分器：
输入 I 型（LF 无积分） II 型（LF 加积分）
相位阶跃 零稳态误差 零
频率阶跃（恒定频差） 残留固定相差 零

三时钟域之间正是恒定频差 → 必须 II 型才能零稳态相差（第五节）。

  • 采样控制稳定性：数字环是采样系统，控制周期 T 越大相位裕度越小越易失稳——本题 5ms 稳、 50ms 发散即此理。
  • PID 离散化： uk=Kpek+Ki∑eT+Kdek−ek−1T ，配抗积分饱和限幅（第五节模拟路线）。

8.7 模拟乘法器鉴相

两同频正弦相乘 → 和频 + 差频，低通滤掉 2ω 留直流：

sin⁡(ωt+φ1)sin⁡(ωt+φ2)=12cos⁡(φ1−φ2)−12cos⁡(2ωt+φ1+φ2)

直流项 ∝cos⁡(Δφ) ，在 Δφ=90∘ 处过零且最线性 → 正交锁定。本题模拟鉴相路线即用此（第五节 B）。

8.8 定点 DSP 与块归一化

  • Q 格式： Q15 把 [−1,1) 映射到整数 [−32768,32767] ；乘法后右移回定标，注意溢出 / 饱和。
  • 块归一化（块浮点）：对整块数据找最大值统一左移、吃满量程再 FFT，事后记下移位——在无 FPU 的 M0+ 上用定点拿到接近浮点的动态范围。
  • 本题用到： q15 定点 FFT（CMSIS-DSP）、块归一化撑起三角波的小谐波（第三、四节）。

九、小结

一句话画像：

硬件：双 ADC 同步采集 → MCU 分析 → DDS 再生 → 闭环锁定 的再生法装置，核心难点是三时钟域漂移逼出来的锁相环。 软件：纯 C 算法核 + 薄硬件胶水 + 离板测试 的可验证三明治；锁相主线走模拟鉴相 + PID，全数字 PLL 作可一键切回的替代。 角色：MSPM0 是大脑，AD9959 是嘴，串口屏是脸，遥测是听诊器。

目前算法层（FFT 分析、波形判别、谐波去污、PID 数学、数字 PLL）都已离板验证；而主线模拟鉴相的"最后一公里"——乘法器、正交本振、 1.65V 偏置、锁定阈值——要上板标定才能最终确认（见 5.2）。把"上电前能锁死的正确性"尽量在电脑上锁死，剩下的交给板子。

下一步是真正的上板联调：先把主线模拟鉴相的前端搭起来、按 5.2 标定，锁不住就先翻环路符号；数字 PLL 留作随时能切回的退路。后续进展再开一篇记录。
